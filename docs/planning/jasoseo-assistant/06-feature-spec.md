# 06. 기능 기획 상세 (MVP 중심)

## 1) 기능 프레임
기능은 아래 4개 묶음으로 제한한다.

1. 작성 시작: 지원 페이지 파싱 + 입력 방식 설정
2. 본문 작성: 생성 + 편집 + 버전 관리
3. 품질 검토: 질문 적합성/근거성/가독성 점검
4. 결과 정리: 최종본 확정/내보내기

## 2) 입력/구조화 요구사항

| 항목 | 요구사항 |
|---|---|
| 지원 페이지 입력 | 사용자가 텍스트를 붙여넣으면 회사/직무/문항/글자수 제한 자동 추출 |
| 형식 처리 | 문항형/단문형/요약형을 별도 메뉴로 강제하지 않고 추출 결과로 통합 처리 |
| 오류 보정 | 자동 추출 결과에서 사용자가 직접 수정 가능 |
| 세션 단위 | 1세션=1지원 공고, 공고 내 문항은 탭으로 동시 편집 |
| 글자수 기준 | 기본 공백 포함 문자수, 공백 제외 토글 지원 |

## 3) 상세 기능 목록

### A. 작성 시작
- `지원 페이지 텍스트 붙여넣기` 입력창
- `요구사항 자동 구조화`: 회사/직무/문항/제약 파싱
- `입력 방식 선택`: 초안 붙여넣기 / 자료 기반 생성

### B. 자료 라이브러리
- 이력서/포트폴리오/기존 자기소개서 업로드 및 선택
- 작성 세션별 참조 자료 연결
- 수동 업로드만 지원 (외부 연동 제외)
- 지원 파일 형식: PDF, DOCX, MD, TXT
- 업로드 용량 제한: 없음
- 스캔 PDF(OCR) 비지원

### C. 작성 에디터
- 중앙 본문 편집
- 좌측 자료 근거 패널
- 우측 `도구 메뉴` + `AI 채팅` 패널
- 도구 메뉴 기본 항목:
  - 다른 소재 찾기
  - 질문 적합성 개선
  - 가독성 개선
  - 분량 맞추기
  - 톤 조정
- 도구 선택 시: 해당 도구 프롬프트를 AI 채팅창으로 자동 전송
- 버전 저장 및 되돌리기
- AI 추론 문장 시각 표시(주황 점선 밑줄 + 경고 배지)
- 수정안 적용 방식
  - 기본: 제안 후 사용자 적용
  - 적용 UX: 기존안(좌) / 수정안(우) 비교, 삭제/추가 하이라이트
  - 옵션: 자동 반영 토글 ON 시 비교 단계 없이 적용
- 적용 범위 제어
  - 선택 문단만 수정(기본)
  - 전체 문서 수정(사용자 선택)
  - 수동 적용 모드: 범위 밖 변경은 별도 제안으로 분리
  - 자동 반영 ON 모드: 범위 밖 변경 허용 (차단 없음)

### D. 검토 리포트
- 질문 대응성 검사 (최우선)
- 근거성 검사 (2순위)
- 가독성 검사 (3순위)
- 근거 부족/AI 추론 문장 표시
- 우선 수정 항목 3개 제시
- 실행 방식: 수동 실행 + 자동 실행(둘 다 지원)
- 자동 검토 유휴 시간: 3초

### E. 내보내기
- 복사/텍스트 출력
- 최종본 스냅샷 보관

### F. 계정/보관
- 인증: 최소 로그인(단일 관리자 계정)
- 보관: 영구 보관
- 삭제: 사용자가 개별/전체 즉시 삭제 가능

## 4) 사실성 처리 규칙
1. 자료에 명시된 사실은 `SUPPORTED` 상태로 생성
2. 자료에는 없지만 있을 법한 내용은 `INFERRED` 상태로만 생성 가능
3. `INFERRED` 문장은 사용자 확인(확정/수정/삭제) 액션 전까지 경고 상태 유지
4. AI는 초안 생성 전후로 누락 가능성이 있는 활동/강점을 사용자에게 질문

## 5) 핵심 사용자 스토리 (MVP)
1. 사용자는 지원 페이지 텍스트를 붙여넣고, 문항 구조를 자동으로 얻을 수 있다.
2. 사용자는 수동 업로드한 자료만으로 특정 회사 문항 초안을 생성할 수 있다.
3. 사용자는 도구 메뉴를 통해 AI 채팅 기반 편집 액션을 빠르게 실행할 수 있다.
4. 사용자는 수정안 비교 뷰에서 변경점을 확인하고 적용 여부를 선택할 수 있다.
5. 사용자는 자동 반영 토글을 켜서 AI 제안을 즉시 반영할 수 있다.
6. 사용자는 근거 없는 AI 추론 문장을 모두 확인/수정한 뒤 최종본을 확정할 수 있다.

## 6) MVP 제외 항목 (후속)
- 팀 협업/멘토 첨삭 기능
- 외부 문서 서비스 연동
- 협업 코멘트/첨삭 워크플로우
- 다국어 자기소개 자동 변환
- 기업/직무별 고급 데이터 연동 기능
