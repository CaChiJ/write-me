# 08. MVP 범위, 지표, 단계별 로드맵

## 1) MVP 범위 정의
MVP는 아래 한 가지 사용자 약속을 달성하는 데 집중한다.

> "개인 사용자가 지원 페이지와 수동 업로드 자료만으로, AI 채팅/도구를 활용해 검증 가능한 자기소개서를 완성한다."

MVP 포함 범위:
- 지원 페이지 텍스트 파싱 기반 요구사항 구조화
- 2가지 시작 방식 지원(초안 편집, 자료 기반 생성)
- 작성 에디터 + 도구 메뉴 + AI 채팅 + 검토 리포트 + 내보내기(복사/텍스트)
- AI 추론 문장 경고 표시 및 사용자 확인 루프
- 제안 적용 비교 뷰(기존안/수정안) + 자동 반영 토글
- 자동 검토 + 수동 검토 병행

## 2) 출시 단계 제안

### Phase 1: 작성 진입/구조화
- 지원 페이지 텍스트 입력
- 회사/문항/글자수 자동 추출
- 초안 붙여넣기/자료 기반 시작
- 버전 저장

### Phase 2: 작성 상호작용 강화
- 도구 메뉴 + AI 채팅 연동
- 도구 클릭 시 프롬프트 자동 전송
- 본문 수정 제안 반영 UX(좌우 비교 diff)
- 적용 범위 선택(선택 문단/전체 문서, 수동 모드에서 범위 잠금)
- 자동 반영 토글
- 자동 반영 ON 시 범위 밖 변경 허용 정책 반영

### Phase 3: 신뢰/검토 강화
- 질문 적합성 > 근거성 > 가독성 검토 리포트
- AI 추론 문장 표시 및 사용자 확인 루프
- 미확정 문장 해소 흐름 최적화
- 자동 검토 트리거 최적화(입력 유휴 기반)

## 3) 핵심 지표 (제품 관점)
- `초안 도달 시간`: 첫 초안 확보까지 걸린 시간
- `완료율`: 작성 시작 대비 최종본 확정 비율
- `수정 반복률`: 검토 후 재수정 비율
- `요구사항 추출 정확도`: 파싱된 문항/제약이 사용자 의도와 일치한 비율
- `미확정 문장 잔존율`: 최종본 확정 시 INFERRED 문장 잔존 비율
- `범위 이탈 수정률`: 지정 범위 밖 텍스트가 수정된 비율(수동 모드 품질 모니터링용)
- `수정안 채택률`: 비교 뷰 제안 중 실제 적용된 비율
- `재방문율`: 다음 공고 지원 시 재사용 비율

## 4) 주요 리스크와 대응
- 리스크: 기능이 많아져 초기 UX 복잡도 상승
  - 대응: 초기 화면에서 3개 시작 버튼만 노출
- 리스크: 생성 문장의 신뢰성 문제
  - 대응: AI 추론 문장 시각 경고 + 사용자 확인 루프
- 리스크: 지원 페이지 파싱 실패/오탐
  - 대응: 자동 추출 결과를 사용자가 즉시 수정 가능하도록 설계

## 5) 즉시 실행 가능한 다음 기획 작업
1. 지원 페이지 파싱 결과 편집 UI 필드 확정
2. 도구 메뉴별 기본 프롬프트 문안 확정
3. AI 추론 문장 경고 UI(색/밑줄/배지) 시안 결정
4. 수정안 비교 뷰(삭제/추가 하이라이트) 상세 인터랙션 정의
5. 자동 검토 재실행 조건(유휴 시간 3초 고정) 정교화
