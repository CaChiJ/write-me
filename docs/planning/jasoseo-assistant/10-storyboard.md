# 10. Storyboard

## 목적
이 문서는 `Write Me`의 MVP 사용 경험을 6프레임 스토리보드로 고정한다.  
이 스토리보드는 감성 정렬용 문서이지만, 화면 설계와 상호작용의 우선순위를 결정하는 기준으로도 사용한다.

## 메인 캐릭터
- 이름: 김민서
- 나이: 25
- 배경: 컴퓨터공학 전공 졸업 예정, 팀 프로젝트 3개와 짧은 인턴 경험 1회 보유
- 목표: 삼성/SK 같은 대기업 문항형 자소서와 스타트업용 짧은 자기소개를 동시에 준비
- 현재 문제: 이력서, 포트폴리오, 기존 자소서가 흩어져 있고, 회사마다 형식이 달라 매번 처음부터 다시 정리해야 함

## 시각 방향
- 렌더링 톤: fat-marker sharpie 스케치가 아니라, 구현 문서에 맞는 `editorial productivity sketch`
- 색감: 라이트 테마, 종이 같은 배경, 따뜻한 중성톤, 강조색은 amber/teal
- 감정선: 초조함 -> 과부하 -> 통제감 상실 -> 신뢰 형성 -> 안도감 -> 제출 자신감

## 6-Frame Storyboard

### Frame 1. 메인 캐릭터 소개
- 장면:
  - 민서는 노트북 앞에 앉아 삼성 지원 페이지, 스타트업 채용 페이지, PDF 이력서, 포트폴리오 문서를 동시에 열어두고 있다.
  - 책상 위에는 메모지와 마감일이 적힌 달력이 널려 있다.
- 감정:
  - 바쁘지만 아직은 통제 가능하다고 느낀다.
- 제품 접점:
  - 아직 제품을 사용하지 않는다.
- 핵심 가치:
  - 사용자는 이미 자료를 갖고 있지만, 그 자료를 회사별 형식에 맞게 재조합하는 데 어려움을 겪는다.
- 스케치 지시문:
  - 왼쪽에는 여러 문서 탭, 오른쪽에는 마감일 스티커, 인물은 약간 앞으로 숙인 자세.

### Frame 2. 문제 발생
- 장면:
  - 삼성 문항형 질문은 700자 제한이고, 스타트업은 4문장 소개만 요구한다.
  - 민서는 같은 경험을 어디까지 줄여야 하는지, 어떤 문장을 어디서 재사용할 수 있는지 모르겠다.
- 감정:
  - 복잡함, 피로감, 형식 전환에서 오는 저항.
- 제품 접점:
  - 아직 제품 없음. 문제는 자료 부족이 아니라 정리 실패임이 드러난다.
- 핵심 가치:
  - 제품은 “글을 대신 써주는 도구”보다 먼저 “지원 형식을 구조화하고 자료를 연결하는 작업대”여야 한다.
- 스케치 지시문:
  - 화면 왼쪽엔 장문의 문항, 오른쪽엔 짧은 소개 요구가 대비되게 배치.

### Frame 3. 오 크랩 모먼트
- 장면:
  - 마감이 오늘 밤인데, 민서는 기존 자소서를 복붙하다가 문항 의도와 안 맞는 문장을 섞어 넣는다.
  - 어떤 성과는 자료에 명시돼 있지 않아 “이 정도는 써도 되나?”라는 불안을 느낀다.
- 감정:
  - 초조함, 자기검열, 신뢰 상실.
- 제품 접점:
  - “근거 없는 문장을 표시해주고, 수정 전후를 비교해주는 도구가 필요하다”는 순간.
- 핵심 가치:
  - MVP의 핵심 신뢰 장치는 `INFERRED 경고`, `diff 비교`, `최종 확정 전 확인`이다.
- 스케치 지시문:
  - 빨간색 느낌의 마감 알림, 복붙된 문장들, 머뭇거리는 커서, 긴장한 표정.

### Frame 4. 해결책 등장
- 장면:
  - 민서는 `Write Me`에 지원 페이지 텍스트를 붙여넣고, 이력서와 포트폴리오를 연결한다.
  - 화면이 회사명, 문항, 글자 수 제한을 자동 구조화하고, 작성 세션을 만든다.
- 감정:
  - 경계심에서 안심으로 넘어가는 시작점.
- 제품 접점:
  - `지원 페이지 붙여넣고 시작`
  - `자료 연결`
  - `문항 탭 자동 구성`
- 핵심 가치:
  - 시작 장벽을 낮추고, 사용자의 기존 자료를 중심에 둔다.
- 스케치 지시문:
  - 중앙에는 붙여넣기 패널, 아래에는 추출된 문항 카드, 우측에는 자료 연결 패널.

### Frame 5. 아하 모먼트
- 장면:
  - 에디터에서 민서는 질문 탭을 넘겨가며 초안을 다듬는다.
  - AI 채팅 패널에서 “2번 문단만 더 간결하게”라고 요청하자, 수정안이 오른쪽에 제시되고 변경점이 비교된다.
  - 근거가 부족한 문장에는 amber 점선 밑줄과 경고 배지가 붙는다.
- 감정:
  - 통제감 회복, 신뢰 형성, 작업 속도 상승.
- 제품 접점:
  - `도구 메뉴`
  - `AI 채팅`
  - `기존안/수정안 diff 비교`
  - `INFERRED 경고`
  - `자동 검토`
- 핵심 가치:
  - AI는 본문을 덮어쓰지 않고, 사용자가 검토 가능한 수정 파트너로 작동해야 한다.
- 스케치 지시문:
  - 3패널 레이아웃, 중앙 본문, 우측 diff/채팅, 주황 점선 밑줄이 보이도록 표현.

### Frame 6. 해결 이후의 삶
- 장면:
  - 민서는 삼성용 문항 답변, 스타트업용 4문장 소개, 그리고 다음 지원 회사용 세션을 정리된 목록으로 보관한다.
  - 리뷰 화면에서 남은 이슈를 모두 해결한 뒤 최종본을 복사해 제출한다.
- 감정:
  - 안도감, 제출 자신감, 재사용 가능한 체계 확보.
- 제품 접점:
  - `최근 작업`
  - `자료 라이브러리`
  - `검토 리포트`
  - `최종본 복사`
- 핵심 가치:
  - 단발성 생성보다 “다음 지원에도 재사용 가능한 자기소개 작업 환경”을 제공한다.
- 스케치 지시문:
  - 정리된 대시보드, 완료 체크된 리뷰 카드, 차분한 표정, 닫힌 노트북 옆 커피잔.

## 스토리보드에서 도출되는 UX 원칙
1. 시작은 빠르고 가벼워야 한다.
2. 자료와 문장은 항상 연결돼야 한다.
3. AI의 변경은 비교 가능해야 한다.
4. 신뢰는 시각적 경고와 확인 루프로 만든다.
5. 사용자는 문항형과 단문형 사이를 같은 작업대에서 이동해야 한다.
6. 제출 직전에는 “무엇이 남았는지”가 한눈에 보여야 한다.

## 화면 설계에 바로 반영할 항목
- 대시보드 최상단에는 `지원 페이지 붙여넣고 시작` CTA를 둔다.
- 새 세션 화면에서는 “붙여넣기 -> 구조화 -> 자료 연결” 순서를 유지한다.
- 메인 에디터는 3패널 구조를 기본으로 한다.
- 수정안은 반드시 기존안/수정안 비교 뷰를 제공한다.
- 검토 화면은 unresolved `INFERRED`와 질문 적합성 이슈를 최상단에 배치한다.
