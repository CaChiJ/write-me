# 01. 제품 범위와 원칙

## 1) 문제 정의
신입 개발자는 자기소개서를 쓸 때 아래 문제가 반복된다.

- 회사마다 요구 형식이 달라 매번 새로 구조를 잡아야 한다.
- 이미 가진 자료(이력서/포트폴리오/기존 자소서)를 재활용하기 어렵다.
- 초안을 만든 뒤에도 논리, 분량, 톤, 질문 적합도를 스스로 점검하기 어렵다.

## 2) 제품 목표
이 서비스의 1차 목표는 다음 3가지다.

- 지원 페이지를 텍스트로 붙여넣으면 형식/문항/제약을 자동 구조화한다.
- "초안 편집"과 "자료 기반 생성"을 같은 작성 세션 안에서 오갈 수 있게 한다.
- 작성부터 검토, 내보내기까지 끊김 없는 흐름을 제공한다.

## 3) 제품 범위 (초기, 확정)
1. **개인 사용자 1명 기준 제품**
   - 팀 협업/멘토 권한 모델 없이 단일 사용자 작성 경험에 집중
2. **지원 형식 통합 처리**
   - 별도 형식 메뉴를 나누기보다, 지원 페이지 텍스트를 분석해 자동 구조화
3. **수동 자료 업로드 고정**
   - 이력서/포트폴리오/기존 자기소개서 업로드 기반
   - 지원 포맷: PDF, DOCX, MD, TXT
   - 업로드 용량 제한 없음
   - 외부 서비스 연동은 초기 범위에서 제외
4. **내보내기 단순화**
   - 복사/텍스트 출력 중심
   - DOCX/PDF 내보내기는 후속
5. **세션 단위 고정**
   - 1세션 = 1지원 공고
   - 공고 내 여러 문항은 한 세션에서 탭 형태로 동시 편집

## 4) 핵심 가치 제안
- `형식 추출 자동화`: 지원 페이지에서 문항 구조와 분량 제약을 추출
- `자료 재활용`: 과거 자료를 근거로 빠르게 초안 생성
- `품질 보조`: 질문 적합성, 근거성, 가독성 점검으로 제출 품질 향상
- `단순 UX`: 복잡한 기능을 숨기고, 작성 중심 동선 유지

## 5) 사실성/신뢰 원칙 (핵심)
- 사용자가 제공하지 않은 사실을 확정 문장으로 단정하지 않는다.
- 다만 "있을 법한 내용"을 AI가 제안할 수는 있으며, 이 경우 반드시 `AI 추론 추가` 상태로 표시한다.
- `AI 추론 추가` 문장은 편집기에서 시각적으로 구분한다.
  - 기본 제안: 주황 점선 밑줄 + 경고 배지 + 툴팁(자료 근거 없음)
- 최종 확정 전, 추론 문장마다 사용자 선택이 필요하다.
  - 선택: 사실로 확정(근거 입력), 문장 수정, 삭제
- 자료에 없는 강점/활동이 있을 수 있으므로, AI는 생성 전후에 보강 질문을 먼저 던진다.
  - 예: "관련 협업 경험이 더 있었나요?", "수치화 가능한 결과가 있나요?"

## 6) Self-hosted 전제에서의 제품 원칙
- 기본 전제는 브라우저 접근 가능한 사내/개인 호스팅 환경
- 민감한 취업 자료(이력서/포트폴리오/자소서)는 외부 공유 없이 내부 보관
- 최소 로그인(단일 관리자 계정) 제공
- 데이터는 영구 보관하되, 사용자가 개별/전체 즉시 삭제 가능
- 설치 후 바로 쓸 수 있는 간결한 초기 설정 경험 제공

## 7) 비목표(초기 단계에서 제외)
- 코딩 테스트/면접 전 범위를 한 번에 다루는 올인원 취업 플랫폼
- 채용 공고 크롤링 기반 자동 지원
- 고급 협업 워크플로우(다중 편집자 실시간 동시 편집 등)
- 외부 계정/문서 서비스 연동 가져오기

## 8) 제품 설계 원칙
1. **먼저 쓰게 한다**: 복잡한 설정 전에 바로 작성 시작 가능해야 함
2. **근거와 문장을 연결한다**: 문장마다 출처 자료를 추적 가능하게 설계
3. **전환 비용을 낮춘다**: 생성 ↔ 편집 ↔ 요약을 같은 화면에서 전환
4. **학습 없이 사용 가능**: 신입 사용자도 5분 내 첫 결과물을 얻어야 함
