# 04. UX 플로우와 와이어프레임

## 1) 핵심 UX 목표
- 사용자가 어떤 시작점이든 3단계 내 첫 초안을 확보
- 작성 화면에서 "근거 확인 ↔ 문장 수정"을 빠르게 반복
- 검토와 수정이 분리되지 않고 한 흐름에서 연결
- AI 추론 문장을 명확히 구분해 신뢰를 유지
- AI 수정안은 기존안과 나란히 비교해 변경점을 직관적으로 확인

## 2) 주요 플로우

### Flow A: 지원 페이지 텍스트 기반 시작 (기본)
1. 대시보드에서 `지원 페이지 붙여넣고 시작` 선택
2. 페이지 텍스트 입력
3. 서비스가 회사/문항/글자수 제한을 구조화해 보여줌
4. 사용자가 필요한 부분만 수정/확정
5. 자료 선택 후 초안 생성
6. 도구 메뉴/AI 채팅으로 개선 후 최종본 확정

### Flow B: 기존 초안 편집 시작
1. 대시보드에서 `기존 초안 붙여넣고 시작` 선택
2. 초안 + 지원 페이지 텍스트(선택)를 입력
3. 문항 탭에서 항목별로 질문 적합성/가독성 개선 제안 확인
4. 검토 리포트 반영 후 완료

### Flow C: AI 추론 문장 확인 루프
1. AI가 자료에 없는 가능성 문장을 제안
2. 문장에 `AI 추론 추가` 경고 표시
3. 사용자는 사실 확정/수정/삭제 중 선택
4. 미확정 문장 해소 후 최종본 확정

### Flow D: 도구 메뉴 기반 수정 적용
1. 사용자가 도구 메뉴 선택 (예: 질문 적합성 개선)
2. 프롬프트가 우측 AI 채팅으로 자동 전송
3. AI가 수정안 생성
4. 시스템이 기존안(좌)/수정안(우) 비교 뷰를 제공
5. 사용자가 변경점 확인 후 `적용` 또는 `취소`
6. 자동 반영 토글 ON이면 비교 뷰 스킵 후 자동 반영 (범위 제한 미적용)

## 3) 저충실도 와이어프레임

### (A) 대시보드
```text
+------------------------------------------------------+
| 로고                              [자료] [설정]      |
+------------------------------------------------------+
| 새 작성 시작                                           |
| [지원 페이지 붙여넣고 시작] [기존 초안 붙여넣고 시작]    |
| [자료 선택 후 시작]                                     |
|                                                      |
| 최근 작업                                              |
| - 삼성 DS 자소서 1번                                   |
| - A 스타트업 500자 소개                                |
+------------------------------------------------------+
```

### (B) 작성 설정(간단 스텝)
```text
+------------------------------------------------------+
| 1. 지원 페이지 텍스트 입력                              |
| [붙여넣기 영역]                                         |
|                                                      |
| 2. 자동 추출 결과 확인                                  |
| 회사명 / 직무 / 문항들 / 글자수 제한                    |
|                                                      |
| 3. 자료 선택(선택)                                     |
| [이력서.pdf] [포트폴리오.md] [기존자소서.docx]          |
|                                              [시작]    |
+------------------------------------------------------+
```

### (C) 작성 에디터
```text
+----------------+-------------------------+---------------------------+
| 근거 자료 패널   | 본문 에디터              | 도구 + AI 채팅               |
| - 이력서 섹션1   | [문항 탭: 1 | 2 | 3]     | [다른 소재 찾기]             |
| - 프로젝트 A     | [작성/편집 영역]          | [질문 적합성 개선]           |
| - 기존 자소서 B  | 글자수: 482/700           | [가독성 개선]                |
|                 | 미확정 문장: 2            | 적용 범위: (선택 문단) 전체  |
|                 | ~~~주황 점선 밑줄~~~      | 자동 반영: OFF               |
|                 | (AI 추론 추가 문장)       | --------------------------- |
|                 |                         | 사용자: 2번 문단만 다듬어줘   |
|                 |                         | AI: 해당 문단 수정안 생성     |
+----------------+-------------------------+---------------------------+
| [검토 실행] [자동 검토 ON/OFF] [버전 저장] [최종본 확정] [미확정 문장 확인] |
+--------------------------------------------------------------------------+
```

### (D) 검토 리포트
```text
+------------------------------------------------------+
| 검토 점수 요약                                         |
| 질문 적합도 8/10 | 근거성 7/10 | 가독성 9/10             |
|                                                      |
| 우선 수정 항목                                        |
| 1) 문항 요구 역량(문제 해결) 직접 언급 부족             |
| 2) 프로젝트 성과 수치 1개 추가 필요                     |
| 3) 마지막 문장 중복 표현 제거                           |
|                                                      |
| 미확정 문장                                            |
| - "배포 자동화를 주도했다" (근거 자료 없음)              |
|                                     [수정 반영하기]     |
+------------------------------------------------------+
```

### (E) 수정안 비교 뷰 (수동 적용 기본)
```text
+------------------------------------------------------------------+
| 수정안 비교                                                       |
| 적용 범위: 2번 문단                                                |
|                                                                  |
| 기존안 (좌)                        | 수정안 (우)                  |
| - 삭제될 문장(빨간 취소선)          | + 추가된 문장(초록 밑줄)      |
|   유지되는 문장(기본색)             |   유지되는 문장(기본색)       |
|                                                                  |
| [취소] [이 수정안만 적용] [다음 제안 보기]                          |
+------------------------------------------------------------------+
```

## 4) UX 디테일 원칙
- "자동 생성" 버튼보다 "현재 문장 개선" 버튼의 노출 우선순위를 높인다.
- 제안은 길게 나열하지 않고 상위 3개만 먼저 보여준다.
- 검토 결과는 추상 점수보다 구체 수정 행동(무엇을, 어디에)을 우선 제시한다.
- 도구 메뉴 클릭 시, 해당 도구 프롬프트를 AI 채팅창에 자동 입력/전송한다.
- 자동 반영 기본값은 OFF로 둔다. 사용자가 토글 ON 시에만 자동 반영한다.
- 수동 적용 모드에서는 지정 범위(선택 문단) 밖 변경을 금지하고 별도 제안으로 분리한다.
- 자동 반영 ON 모드에서는 범위 밖 변경도 허용한다.
- 자동 검토는 입력 후 3초 유휴 시 실행하되, 수동 검토 버튼은 항상 유지한다.
