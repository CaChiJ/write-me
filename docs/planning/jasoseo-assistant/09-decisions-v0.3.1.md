# 09. 확정 결정사항 (v0.3.1)

## 반영 기준
- 확인 일시: 2026-03-11
- 결정 주체: 사용자 확인 응답 기반
- 목적: MVP 범위, 편집 UX, 운영 정책 고정

## 제품 범위 결정
1. **사용자 모델**
   - 개인 사용자 1명 기준
2. **세션 단위**
   - 1세션 = 1지원 공고
3. **문항 편집 방식**
   - 한 공고 내 복수 문항을 한 화면 탭에서 동시 편집
4. **지원 페이지 입력**
   - 텍스트 붙여넣기만 지원
5. **파싱 보정 권한**
   - 회사/직무/문항/글자수 제한 모두 사용자 수정 가능
6. **형식 처리**
   - 형식별 복잡한 분기 UI 대신 자동 구조화 중심

## 사실성/신뢰 결정
1. **AI 추론 문장 정책**
   - `INFERRED` 문장 허용
   - 최종 확정 전 전부 사용자 확인 필수
2. **INFERRED 처리 단위**
   - 문장별 처리만 허용(일괄 처리 없음)
3. **누락 정보 질문**
   - AI가 강점/활동 누락 가능성을 사용자에게 보강 질문

## 편집 UX 결정
1. **수정 적용 기본 모드**
   - 제안 후 사용자 적용
2. **비교 방식**
   - 기존안(좌) / 수정안(우) 나란히 표시
   - 삭제/추가 내용을 직관적으로 하이라이트
3. **자동 반영 옵션**
   - 사용자 토글 ON 시 자동 반영 허용
4. **적용 범위**
   - 선택 문단/전체 문서 모두 지원
   - 수동 적용 모드: 범위 밖 수정 방지 프롬프트 가드레일 적용
   - 자동 반영 ON 모드: 범위 밖 수정 허용
5. **검토 실행 방식**
   - 수동 + 자동 모두 지원
6. **자동 검토 유휴 시간**
   - 3초로 고정

## 문서/운영 정책 결정
1. **업로드 포맷**
   - PDF, DOCX, MD, TXT
   - 업로드 용량 제한 없음
2. **스캔 PDF**
   - 비지원(입력된다고 가정하지 않음)
3. **보관 정책**
   - 영구 보관
   - 사용자 즉시 삭제 기능 제공
4. **인증 정책**
   - 최소 로그인(단일 관리자 계정)

## 메모
- 자동 반영 ON 모드에서 광범위 수정이 발생할 수 있으므로, 변경 이력 확인 UX를 기본 제공한다.
