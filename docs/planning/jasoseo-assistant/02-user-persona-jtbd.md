# 02. 사용자 유형과 핵심 시나리오

## 1) 주요 사용자 유형

### Persona: 개인 사용자(신입 개발자) 1명
- 목표: 회사별 요구가 달라도 빠르게 제출 가능한 자기소개서를 완성
- 특징: 다양한 형식을 매번 해석하기 어렵고, 근거 있는 문장 작성이 부담됨
- 기대: 지원 페이지를 넣으면 구조를 알아서 정리하고, AI와 대화하며 문장을 개선

## 2) 핵심 JTBD (Jobs To Be Done)
1. "지원 페이지의 요구사항을 매번 직접 구조화하지 않고, 빠르게 작성을 시작하고 싶다."
2. "내가 가진 자료를 다시 쓰지 않고 활용해, 품질 높은 초안을 얻고 싶다."
3. "내 문장이 질문 의도에 맞는지, 그리고 근거가 충분한지 객관적으로 검토받고 싶다."

## 3) 대표 사용 시나리오

### 시나리오 1: 지원 페이지 텍스트 기반 시작
- 사용자가 지원 페이지의 텍스트를 붙여넣음
- 서비스가 회사/문항/글자수 제한을 자동 추출해 작성 세션 생성
- 사용자가 업로드 자료를 연결하고 초안을 생성
- AI 채팅과 도구 메뉴로 문장 개선 후 최종본 확정

### 시나리오 2: 기존 초안 편집 시작
- 사용자가 자기소개서 초안을 붙여넣음
- 서비스가 지원 페이지 요구사항과 초안을 매핑해 부족 항목을 표시
- 사용자가 도구 메뉴(질문 적합성 개선, 가독성 개선 등)로 수정 반복

### 시나리오 3: 하나의 세션에서 혼합 작업
- 처음에는 자료 기반으로 초안 생성
- AI가 "자료에 없지만 있을 법한 내용"을 제안해 문장 추가
- 해당 문장이 경고 표시되며, 사용자가 사실 여부를 확정/수정/삭제

## 4) 사용자 성공 기준
- 첫 작성 결과물이 10분 내 확보됨
- 목표 분량 오차가 작음(예: 제한 대비 ±10% 이내)
- 검토 우선순위(질문 적합성 > 근거성 > 가독성) 기준으로 수정 완료
- 최종본에 근거 미확정 문장이 남지 않거나, 남아도 사용자가 명시적으로 승인
