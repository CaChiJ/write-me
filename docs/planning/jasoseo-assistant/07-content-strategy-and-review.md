# 07. 콘텐츠 품질/가이드 전략

## 1) 품질 기준 (초기 버전)
자기소개서 품질을 아래 5가지로 정의한다.

1. `질문 적합성`: 질문이 요구한 역량/경험에 직접 답하는가
2. `근거성`: 이력서/포트폴리오 등 자료와 모순되지 않는가
3. `가독성`: 문장이 짧고 명확하며 중복이 적은가
4. `구체성`: 추상 표현보다 실제 행동/성과가 포함되는가
5. `일관성`: 문단 간 톤과 메시지가 충돌하지 않는가

## 2) 검토 우선순위 (확정)
1. 질문 적합성
2. 근거성
3. 가독성

## 3) 사실성 정책 상세
- 기본 원칙: 사용자 자료에 없는 사실은 확정 문장으로 만들지 않는다.
- 예외적 제안: AI가 추론으로 문장을 제안할 수 있으나 `INFERRED`로 표시한다.
- `INFERRED` 문장은 시각 표시를 유지한다.
  - 주황 점선 밑줄
  - 경고 배지(자료 근거 없음)
  - 마우스 오버 시 설명 툴팁
- 최종본 확정 전, 사용자는 모든 `INFERRED` 문장에 대해 확정/수정/삭제를 수행해야 한다.

## 4) 누락 정보 보강 질문 전략
- AI는 작성 전후에 누락 가능성이 있는 활동/강점을 질문한다.
- 질문 예시:
  - "문제 해결 경험에서 수치로 말할 수 있는 결과가 있나요?"
  - "협업 갈등을 조율한 사례가 있나요?"
  - "지원 직무와 직접 연결되는 프로젝트가 더 있나요?"
- 사용자가 추가 사실을 제공하면 해당 문장 상태를 `SUPPORTED`로 전환한다.

## 5) 검토 결과의 표현 원칙
- 점수만 보여주지 않고 "수정 행동"으로 번역해 제시
- 한 번에 너무 많은 피드백을 주지 않고 상위 3개 우선 노출
- 사용자에게 선택권 제공: 기본은 항목별 적용, 필요 시 자동 반영 토글 허용

## 6) 프롬프트/수정 가드레일
- 범위 우선(수동 적용 모드): 사용자가 지정한 범위(문단/문항) 밖 텍스트는 수정 금지
- 자동 반영 ON 모드: 범위 밖 수정도 허용
- 변경 최소화: 요청과 무관한 어휘 치환/문장 순서 변경 금지
- 변경 이유 명시: 수정 제안마다 "왜 바꿨는지" 짧게 설명
- 수동 적용 모드에서 범위 밖 개선 필요 시: 본문 직접 수정 대신 별도 제안 카드로 노출

## 7) 사용자 학습 루프
- 검토 이력에서 반복 지적 항목을 보여줌
- 사용자별 자주 수정하는 패턴(길이 과다, 추상어 남용 등)을 요약
- 다음 작성 세션 시작 시 맞춤 가이드로 선반영
